# Dynamic OpenAPI Discovery

OpenAPI specs generated by services must currently be collected by the API gateway at build time.

Ideally, services should provide the specs to the API gateway at runtime to reduce deployment dependencies.

# Domain Service to Microservice

This is not necessary, but just something fun I'd like to learn how to do.

Service endpoints are annotated with the `#[api(...)]` macro. This means that we explicitly define what functions will be callable via the API gateway, and allows us to create independently scalable functions a la lambda.

The approach I'm currently most interested in is compiling these functions to web assembly and running WASM runtimes in a Kubernetes cluster. This would allow for very small images that are easily autoscaled.

As a more practical stepping stone, using the standard approach we use for domain images would also work, albeit being a bit large.
